PR-02: Add QA spec + generator integration for producing prompt2flow.json
Goal

When cards2pack generate --prompt is used:

generate prompt2flow.json via greentic-qa (interactive)

or via --prompt-json answers file (non-interactive)

write prompt2flow.json into the generated pack assets/config

CLI

Add flags:

--prompt

--prompt-json <path>

Rules:

error if --prompt-json without --prompt

if --prompt and no json: run interactive QA

if --prompt and json: run QA using provided answers

QA spec

Add a greentic-qa questionnaire definition (whatever format you use) that outputs the final JSON shape used by the component. Keep bounded repeats if needed:

max intents = 50

max examples per intent = 20

stop early when user selects “done” or submits empty string

Files

qa/prompt2flow.qa.yaml (or equivalent)

src/qa_integration.rs (thin wrapper: run QA → produce JSON blob)

src/cli.rs changes for flags

src/generate.rs changes to call QA when requested and write output file

Acceptance

cards2pack generate --prompt creates a pack containing prompt2flow.json

cards2pack generate --prompt --prompt-json ./answers.json produces deterministic output (same JSON)
